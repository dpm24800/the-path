# **Super Key**

A **Super Key** is a **set of one or more columns that can uniquely identify each row in a table**.

**Key Points:**
* Every **primary key, candidate key, and alternate key** is a super key.
* But not every super key is minimal.
* If a super key has **extra columns** that are not necessary to maintain uniqueness, it is **not a candidate key**.
* In other words: **Candidate Key = Minimal Super Key**.

**Example:**

```sql
CREATE TABLE Employees (
    EmployeeID INT,
    Email VARCHAR(100) NOT NULL,
    NationalID VARCHAR(20) NOT NULL,
    Name VARCHAR(100)
);
```

**Super Keys examples:**
  1. `(EmployeeID)`
  2. `(Email)`
  3. `(NationalID)`
  4. `(EmployeeID, Email)` – this is still unique, so also a super key, but **not minimal**
  5. `(EmployeeID, Name)` – still unique, also a super key, but extra column makes it non-minimal

**Candidate Keys:** EmployeeID, Email, NationalID (minimal super keys)
**Primary Key:** EmployeeID
**Alternate Keys:** Email, NationalID

## **Relationship Summary:**
- **Super Key:** Any column(s) that uniquely identifies rows (may have extra columns).
- **Candidate Key:** Minimal super key (no extra columns).
- **Primary Key:** Chosen candidate key.
- **Alternate Key:** Candidate key not chosen as primary.
- **Unique Key:** Enforces uniqueness (can be candidate/alternate).
- **Foreign Key:** References primary key of another table.
- **Composite Key:** Primary key made of multiple columns.
